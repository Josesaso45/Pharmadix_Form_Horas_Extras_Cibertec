App Android nativa (sin Jetpack Compose)

Esta sección resume la guía para construir una aplicación Android nativa basada en Views XML, alineada con la arquitectura y los procesos documentados para Pharmadix Times.

Objetivo

Implementar una app Android para Tomadores de Tiempos, Supervisores y Administradores que:

  • Gestione hojas de tiempo y lotes de producción.
  • Permita el registro de operarios mediante escaneo de códigos QR.
  • Sincronice datos con el backend Fastify/Node.js existente.
  • Respete los principios de trazabilidad y ALCOA+ definidos en la arquitectura técnica.

Arquitectura propuesta

La arquitectura Android replica los conceptos clave de la PWA:

  • Capa de Presentación: Activities y Fragments con layouts XML y Navigation Component.
  • Capa de Datos: Room (SQLite) como caché local y Retrofit/OkHttp para la API REST.
  • Capa de Dominio (opcional): Casos de uso para operaciones como crear hoja, registrar entrada/salida, cerrar y aprobar hojas.

El modelo de datos local debe mantenerse coherente con Arquitectura_Diseno_Tecnico_v2.md y Flujo_Procesos_Pharmadix.md.

Flujo de pantallas

Los flujos se inspiran directamente en los diagramas de procesos ya definidos:

  • LoginActivity: autenticación contra la API.
  • Dashboard: acceso a “Nueva Hoja”, “Mis Hojas” y estado de conexión.
  • NuevaHojaFragment: alta de hoja de tiempo para un lote.
  • RegistroOperariosFragment: escaneo de QR y registro de entrada/salida.
  • MisHojasFragment y DetalleHojaFragment: gestión de hojas, cierre y doble confirmación.
